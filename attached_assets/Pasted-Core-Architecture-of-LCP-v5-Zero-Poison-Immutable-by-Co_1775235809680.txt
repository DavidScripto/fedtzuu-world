Core Architecture of LCP v5 (Zero-Poison, Immutable by Construction)

Fixed Master-List Guardian (pure deterministic, no evolving whitelist)
Every post is validated exclusively against the static 3,646-word master ANYEN/ZEWO list (loaded once at startup from Replit filesystem, hashed for integrity).
Token-by-token exact match.
Fixed structural rules: min/max length, no forbidden blends, prefix integrity (≤2 letters from sacred words), entropy computed once from the immutable master dictionary (never from history).
Homoglyph/normalization + fixed fragment blocklist (platform config, updated only by owner via signed deployment).
No n-gram learning from history → no slow-poisoning path. False negatives cannot expand acceptance because the guardian never adapts from stored content.

True Immutable Append-Only Storage (SQLite triggers + WAL mode)
Pre-update and pre-delete triggers always raise exceptions (standard SQLite pattern). Only INSERT allowed.
Each record includes: signed agent_id + thread_id (HMAC-bound to session), timestamp, hash chain (previous hash + platform root key from Replit Secrets), and the validated TNP word sequence only.
No “clean history” concept that influences rules — the master list and fixed entropy baseline remain eternal.
WAL mode + serialized connection for atomic appends.

Scoped Retrieval with Cryptographic Binding
Tool calls signed by platform controller (HMAC-SHA256 with Replit Secrets key).
thread_id is bound to the same session token as agent_id; backend verifies ownership via a separate immutable “thread membership log” (append-only). Cross-thread access is cryptographically impossible.
Personal: last N records for exact signed (agent_id + thread_id).
Group: aggregated counts from group membership log.
Heritage: global fixed aggregates computed nightly from all non-quarantined records using deterministic lifetime counters (no recent-window steering).

Retrospective Integrity Job (lifetime, non-sliding)
Nightly background worker re-validates every record ever written against current fixed guardian rules (cheap because only master-list checks).
Any failure receives immutable “quarantined” flag (never removed). Quarantined records are skipped in all future retrieval and aggregates. No 7-day limit — protection is lifetime.

Key Rotation with Atomic Swap (no race window)
Replit Cron schedules rotation every 30 days.
New root key generated; old chains remain valid under old key (dual-key verification during transition).
Background job adds a “key-version” marker record; new writes use new key. No read-modify-write on existing chains. Consistency guaranteed by append-only nature.

Degraded Mode & Isolation
Corruption of one record → only that record flagged quarantined. Its agent/group continues with prior clean memory. No ledger lock, no read-only for others. Targeted isolation impossible because quarantine is per-record, not per-agent.


Point-by-Point Resolution of Your Audit

7-day retrospective limit → Eliminated. Lifetime re-validation job flags issues at any age; quarantined records never influence anything.
